#### Changes Made

Removed custom className from the "Mark as Sent" Button component:

**Before:**
```tsx
<Button
  type="button"
  onClick={handleMarkAsSent}
  disabled={updateClimb.isPending}
  variant="outline"
  className="w-full border-emerald-500/50 hover:bg-emerald-500/10 text-emerald-400 hover:text-emerald-300"
>
```

**After:**
```tsx
<Button
  type="button"
  onClick={handleMarkAsSent}
  disabled={updateClimb.isPending}
  variant="outline"
>
```

#### Rationale

The custom className was manually overriding shadcn/ui Button's outline variant styling, creating:
- White background (from border-emerald-500/50)
- Custom emerald color overrides that didn't match the app's design system
- Inconsistent appearance compared to other buttons in the app

By removing the custom className, the Button now uses shadcn/ui's built-in outline variant which:
- Properly inherits the app's design tokens
- Provides consistent hover states
- Uses the correct background color from the theme
- Follows shadcn/ui patterns used throughout the codebase
